## Design Context

### Users
This product is for a solo operator who wants a private, self-controlled temporary mail tool. The core job is to create new inbox addresses instantly, monitor incoming mail across many aliases, and keep the whole system lightweight enough to run on managed free tiers.

### Brand Personality
Sharp, operational, premium. The interface should feel like a private control room rather than a generic admin panel or consumer inbox.

### Aesthetic Direction
Editorial control-deck UI with warm neutrals, dark ink surfaces, and a restrained signal accent. The layout should feel deliberate and slightly high-ceremony, but still fast and practical for day-to-day use.

### Design Principles
1. Prioritize fast operational scanning over decorative clutter.
2. Give every address and every inbox a sense of identity without turning the UI playful or noisy.
3. Keep the primary workflow one click away: create address, open inbox, inspect message.
4. Use motion sparingly, mostly for staged reveals and state changes.
5. Avoid the generic SaaS dashboard look; favor stronger typography and layout rhythm.
